When programming, it can be helpful to have something that lets a program decide on a course of action based upon user input or some other criteria. For example, say you want to add numbers onto a running total, but only if the number entered is greater than 5.

This is where conditionals come in. Conditionals evaluate the truth of a statement. For example, let's think about the statement "x > 5". If x is greater than 5, meaning the statement is true, the statement(s) after the conditional is executed. If x is less than or equal to 5, then the statement is false and the statement(s) after the conditional is skipped over.

In most programming languages, we write conditionals in a form similar to this:

if statement1
then action1
else if statement2
then action2
else action3

In this case, your program will perform action1 when statement1 is true. If statement1 isn't true, then your program will perform action2 if statement2 is true and action3 otherwise.

Remember that statements look like the example we did above: stuff like "x > 5", "number of people > number of cars", and "my name is longer than your name".

You can have conditionals which look a little different than the example above. If you don't want to do anything if a statement is false, you could just have an if check without any else's. And you can have any number of "else if" statements (or none at all) to handle any number of different cases.

Conditionals can also be used to decide which path to take when something is not true.
The exclamation mark, !, is the symbol for not and usually appears before the expression it is negating. For example:
If x is true, then !x is false. If y = 25, y > 50 is false. !(y > 50) is true.
